- (direction) User chose Facilitator mode and a 3-technique batch: SCAMPER, Mind Mapping, How Might We
- (technique) started SCAMPER Method
- (idea) MVP expense entry requires only amount, category, and date; all other fields optional
- (idea) User-managed categories: a few sensible defaults (Food, Transport, Shopping, Bills, Entertainment) plus add/edit/delete for custom categories
- (idea) Combine with monthly budget tracker: per-category budget limits with spent-vs-remaining view
- (idea) Adapt fitness-app streaks: logging-streak counter for consecutive days of expense entry to build the habit
- (idea) Visual dashboard: summary cards (Today's Spending, This Month's Spending, Remaining Budget) + pie chart by category + bar chart by month, with detailed tables still available
- (idea) Reuse expense data for monthly insights: top spending categories, month-over-month comparison, and save-money suggestions
- (idea) Eliminate for v1: bank account sync and receipt photo uploads; keep entry fully manual to focus on core features
- (idea) Reverse the flow: evening daily reminder notification prompting the user to log today's expenses, reinforcing the streak
- (technique) started Mind Mapping
- (insight) Mind map top-level branches: Expenses (CRUD), Categories, Dashboard, Budget
- (idea) Expenses branch children: Add, Edit, Delete, View, Search, Filter
- (idea) Filter node children: filter by date range, category, and amount range; search by expense description
- (idea) Budget node children: set monthly budget, view spent amount, view remaining, and within/over-budget status
- (idea) Over-budget signal: budget summary card turns red with a warning message when spending exceeds the monthly budget
- (idea) Categories node children: name + optional icon per category; default set plus fully custom categories
- (technique) started How Might We
- (question) HMW make expense tracking simple and quick so users consistently record daily expenses and understand their spending without feeling overwhelmed?
- (question) HMW help users stay within their monthly budget by giving clear, timely feedback about their spending?
- (question) HMW encourage users to consistently log daily expenses by making the experience simple, rewarding, and easy to maintain over time?
- (decision) Chosen sharpest HMW to ideate against: make expense tracking simple and quick, since consistent data entry is the foundation everything else (dashboard, budget) depends on
- (idea) Smart defaults on the entry form: auto-select today's date, pre-fill last-used category, to minimize fields the user has to touch
- (idea) Duplicate a past expense (one click, just update the date) instead of building a full recurring-transactions engine
- (idea) Persistent floating 'Add Expense' button visible on every page for one-click access to the entry form
- (technique) started MoSCoW convergence
- (decision) MoSCoW - Entry & Data: Must=minimal required fields (amount/category/date), search/filter; Should=smart defaults; Could=duplicate expense; Won't=none
- (decision) MoSCoW - Categories: Must=default categories + add/edit/delete; Could=optional icons; Won't=none
- (decision) MoSCoW - Budget: Must=set monthly budget + spent/remaining/status view; Should=red over-budget warning card; Won't=none
- (decision) MoSCoW - Dashboard: Must=summary cards + detail table; Should=pie chart + bar chart; Could=monthly insights (top categories, MoM comparison, save-money suggestions); Won't=none
- (decision) MoSCoW - Engagement: Must=floating Add Expense button; Could=daily logging streak counter; Won't=evening reminder notification
- (decision) MoSCoW complete across all groups; bank sync and receipt photo uploads remain out of scope (Won't) as decided during SCAMPER Eliminate
- (insight) Synthesis: MVP = simple/fast/useful core CRUD+budget+dashboard; user deliberately chose manual-but-fast over automatic-but-complex (cut bank sync, receipts, reminders; kept duplicate, smart defaults, streak). Streak+duplicate+defaults are one consistency-focused theme via a pull (not push) engagement model. Over-budget red card and dashboard summary cards are the same reusable component with a color-state prop. Monthly insights (Could-have) reuses the same aggregation queries the MVP dashboard already needs, making it a cheap first V2 pick. Chosen direction: build MVP per MoSCoW Must-haves first; Should/Could items become V2 roadmap; Won't items stay explicitly out of scope.
