TECHNIQUE 01

SCAMPER seven lenses on one idea

Each lens turned the same core concept — a simple expense tracker — a little differently, surfacing a feature or a deliberate cut. Click a lens to reveal what it found.

S Substitute

MVP expense entry requires only amount, category, and date; all other fields are optional — substituting a long form for the bare minimum needed to log a spend.

C Combine

Combine the tracker with a monthly budget tracker: per-category budget limits with a spent-vs-remaining view.

A Adapt

Adapt the fitness-app streak mechanic: a logging-streak counter for consecutive days of expense entry, to build the habit of daily logging.

M Modify

Modify the plain data view into a visual dashboard: summary cards (Today's Spending, This Month's Spending, Remaining Budget) plus a pie chart by category and a bar chart by month — with detailed tables still available underneath.

P Put to other use

Reuse expense data for monthly insights: top spending categories, month-over-month comparison, and save-money suggestions — the same records, put to a new purpose.

E Eliminate

Eliminate for v1: bank account sync and receipt photo uploads — keep entry fully manual so the team can focus on the core features first.

R Reverse

Reverse the flow: instead of the user chasing the app, an evening daily reminder notification prompts the user to log today's expenses, reinforcing the streak.

TECHNIQUE 03

How Might We three questions, one chosen

Three opportunity questions surfaced. The sharpest one was chosen as the foundation everything else depends on, and ideated against directly.

SMART DEFAULTS Auto-select today's date and pre-fill the last-used category, minimizing the fields the user must touch.
DUPLICATE EXPENSE One click duplicates a past expense — just update the date — instead of building a full recurring-transactions engine.
FLOATING ADD BUTTON A persistent floating "Add Expense" button, visible on every page, for one-click access to the entry form.

Synthesis & Direction

Where the four techniques converged — and what gets built first
ONE ENGAGEMENT THEME Streak + duplicate expense + smart defaults are one consistency-focused thread: a pull, not a push, engagement model — the reminder notification (a push) was deliberately left out.
ONE REUSABLE COMPONENT The over-budget red warning card (Mind Mapping) and the dashboard summary cards (SCAMPER Modify) are the same component with a color-state prop.
ONE CHEAP V2 WIN Monthly insights (Could-have) reuse the same aggregation queries the MVP dashboard already needs — making it the cheapest first V2 pick.
ONE DELIBERATE BOUNDARY Bank sync, receipt uploads, and reminder notifications stay explicitly out of scope — manual-but-fast was chosen over automatic-but-complex.
CHOSEN DIRECTION

MVP = simple, fast, useful core: CRUD + budget + dashboard, built strictly from the MoSCoW Must-haves first. The user deliberately chose manual-but-fast over automatic-but-complex.

Should- and Could-have items become the V2 roadmap. Won't-have items (bank sync, receipt uploads, reminder notifications) remain explicitly out of scope.

①

MVP — MUST

Minimal entry fields, search/filter, default categories with add/edit/delete, monthly budget with status view, dashboard summary cards + table, floating Add Expense button.

②

V2 — SHOULD / COULD

Smart defaults, red over-budget card, pie/bar charts, duplicate expense, category icons, monthly insights, logging streak.

③

OUT OF SCOPE — WON'T

Bank account sync, receipt photo uploads, evening reminder notifications.
